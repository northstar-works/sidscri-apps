🔧 Quick Actions
📡 Remote Server Config (Push to Android Apps)
Set the active server host and port. Android apps call /api/sync/remote-config on startup and will automatically switch to the configured server. Default: sidscri.tplinkdns.com:8009.
Domain name or IP address (no http:// prefix)
ℹ️ How Android Apps Use This Config
On startup: The Android app calls GET /api/sync/remote-config on its currently configured server. If the returned host or port differs from what the app has saved locally, the app will prompt the user to switch servers (or auto-switch if enabled).

Server Types:
• Standalone – Python Flask server running directly (dev or manual deploy)
• Packaged – Windows EXE installer version
• Raspberry Pi – RPi-specific deployment package
• Docker Container – Running in a Docker container (auto-fills 192.168.0.172:8009)

No push notifications required – the app polls this endpoint on each login/startup.
